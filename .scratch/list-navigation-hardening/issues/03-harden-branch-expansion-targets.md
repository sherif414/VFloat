## What to build

## Agent Brief

**Category:** bug
**Summary:** Harden nested branch expansion to target the first enabled descendant and stay on opener if none exist

**Current behavior:**
When a branch is expanded (using ArrowRight or ArrowLeft in RTL), `useListNavigation` calls `collection.expandBranch(activeValue)` followed by `collection.setNext()`. Because `setNext()` blindly moves to the next focusable item in the flattened items list, if all descendants of that branch are disabled, focus escapes the branch and jumps to an unrelated sibling or root item. If the immediate child is disabled, it will skip it, but may still escape the branch if there are no other enabled descendants in it.

**Desired behavior:**
When a branch is expanded, the selection must target the first enabled descendant _within that expanded branch_.
If the branch has no enabled descendants, the active selection must remain on the branch opener (the parent item), preventing focus from jumping to unrelated siblings or root items.
Collapse behavior (ArrowLeft or ArrowRight in RTL) must return to the parent branch item and collapse it cleanly.
RTL expand/collapse keys must be fully respected and reversed consistently.

**Key interfaces:**

- `useCollection` in `use-collection.ts` — may benefit from dedicated or internal traversal helpers to find descendants of a branch or the first enabled descendant inside a branch, without blindly advancing the global collection index.
- `useListNavigation` branch expand logic inside `onFloatingKeyDown` — needs to target the first enabled descendant of the expanded branch specifically.

**Acceptance criteria:**

- [x] Expanding a branch with enabled descendants moves the active item to the first enabled descendant inside that branch.
- [x] Expanding a branch whose immediate child is disabled skips it and finds the next enabled descendant _within the branch_.
- [x] Expanding a branch with only disabled descendants (or empty/no descendants) leaves `activeValue` on the opener (does not jump to any sibling/root item).
- [x] Collapse behavior collapses the branch and returns `activeValue` to the parent branch opener.
- [x] RTL layout reverses expand/collapse keys (ArrowLeft expands, ArrowRight collapses) consistently.
- [x] Traversal and branch targeting are verified with unit tests in `use-collection.test.ts` and `use-list-navigation.test.ts`.

**Out of scope:**

- Modifying broad tree structure/indexing behavior unrelated to active-value traversal.
